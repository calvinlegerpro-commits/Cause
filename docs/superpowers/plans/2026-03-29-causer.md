# Causer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Forker Parler en Causer — rebrand, interface française par défaut, settings simplifiés Basic/Avancé, wizard d'onboarding clair, et page de distribution GitHub Pages.

**Architecture:** Fork léger de Melvynx/Parler (Tauri + React + Rust). On ne touche pas au moteur Rust ni aux providers IA. Les changements sont concentrés sur : config Tauri, i18n par défaut, réorganisation UI des settings, amélioration du wizard existant, workflow release, et ajout d'une page GitHub Pages statique.

**Tech Stack:** Tauri 2, React 18, TypeScript, Tailwind CSS, react-i18next (fr locale déjà présente), GitHub Actions, HTML/CSS statique pour GitHub Pages.

## Task 3 : Français comme langue par défaut

**Files:**
- Modify: `src/i18n/index.ts`
- Modify: `src/i18n/locales/fr/` (vérification et complétion des traductions)

**Context :** Parler détecte la langue système et tombe sur `en` par défaut si non reconnue. On force `fr` comme fallback principal. La locale `fr` existe déjà — on la complète si nécessaire.

- [ ] **Step 1 : Lire `src/i18n/index.ts`**

  ```bash
  cat src/i18n/index.ts
  ```

  Identifier le champ `lng` ou `fallbackLng` dans la config i18next.

- [ ] **Step 2 : Forcer le français par défaut**

  Dans `src/i18n/index.ts`, modifier la config i18next pour que le français soit la langue par défaut si la langue système n'est pas reconnue :

  ```typescript
  i18n
    .use(initReactI18next)
    .init({
      resources,
      lng: detectBrowserLanguage() ?? 'fr',  // fr au lieu de 'en'
      fallbackLng: 'fr',                      // fr au lieu de 'en'
      interpolation: {
        escapeValue: false,
      },
    });
  ```

  Si la structure est différente, adapter en remplaçant la valeur `fallbackLng: 'en'` par `fallbackLng: 'fr'`.

- [ ] **Step 3 : Vérifier les traductions françaises manquantes**

  ```bash
  # Lister les clés dans en/ et fr/ pour trouver les manquantes
  ls src/i18n/locales/en/
  ls src/i18n/locales/fr/
  ```

  Pour chaque fichier JSON présent en `en/` mais absent ou incomplet en `fr/` : ouvrir le fichier anglais et traduire les valeurs en français. Les clés (keys) restent en anglais — seules les valeurs changent.

  Exemple : si `src/i18n/locales/en/common.json` contient :
  ```json
  { "save": "Save", "cancel": "Cancel", "settings": "Settings" }
  ```
  Alors `src/i18n/locales/fr/common.json` doit contenir :
  ```json
  { "save": "Enregistrer", "cancel": "Annuler", "settings": "Paramètres" }
  ```

- [ ] **Step 4 : Vérifier dans l'app**

  ```bash
  pnpm tauri dev
  ```

  Attendu : toute l'interface est en français. Si certains textes restent en anglais, identifier leur clé i18n et les ajouter à la locale `fr`.

- [ ] **Step 5 : Commit**

  ```bash
  git add src/i18n/
  git commit -m "feat: français comme langue par défaut, complétion locale fr"
  ```
